#1 Trabajo en PP

Resumen de lo aprendido

19 May 2026 - PP Session

Aprendimos que Git no es solo para subir archivos, sino para ordenar el trabajo, colaborar mejor y recuperar el control del proyecto. Entendimos que cada comando cumple una funcion concreta: git add prepara cambios, git commit guarda una version con sentido y git push la publica en el repositorio remoto. Tambien vimos que revisar el origen con git remote -v y traer cambios con git pull evita desincronizaciones.

Tambien entendimos por que aparecio el error 403 al intentar hacer git push. Aunque estabamos trabajando juntos con Live Share, GitHub seguia validando la identidad de la cuenta que estaba autenticada en esa terminal. Como el repositorio original pertenecia a otra cuenta, el servidor rechazo el envio por falta de permisos. Eso nos mostro que compartir edicion o terminal no significa compartir automaticamente credenciales, accesos ni autorizaciones de GitHub.

Los errores de terminal tambien sirvieron para aprender. Varias veces el problema no era Git, sino el contexto: estar en una carpeta distinta, usar una ruta incorrecta, intentar agregar un archivo desde el directorio equivocado o trabajar con ramas y origenes que no coincidian. Con Live Share vimos que colaborar en tiempo real acelera mucho el aprendizaje, pero tambien exige mas atencion sobre quien ejecuta los comandos, desde donde lo hace y con que cuenta esta conectado.

Del lado del desarrollo web, reforzamos la base de HTML, el uso de Emmet para escribir mas rapido, la importancia de una estructura limpia y el valor de probar en un servidor local. En conjunto, lo mas importante fue entender que programar mejor no depende solo de escribir codigo, sino tambien de saber organizarse, corregir, compartir avances y trabajar en equipo.

Tecnologias y herramientas usadas

HTML5, CSS3, JavaScript, Git, GitHub, Visual Studio Code, Live Share, Emmet, terminal de Windows, servidor local y port forwarding para compartir cambios en tiempo real.
