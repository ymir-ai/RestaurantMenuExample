Actividad junior 1

Maqueta un login con Tailwind CSS

Usa este archivo como punto de partida. La idea es que tu companero complete las clases de Tailwind para lograr un login centrado, con un logo arriba, labels a la izquierda, inputs a la derecha y un boton de ingreso bien visible.

Checklist

  • Agregar un fondo suave que ocupe toda la pantalla.
  • Centrar la tarjeta de login vertical y horizontalmente.
  • Usar al menos 4 clases rounded en elementos visibles.
  • Dejar el logo como primer elemento visual del form.
  • Ordenar cada fila con label a un lado e input al otro.
  • Estilizar el boton con hover, escala y transicion.
  • Evitar overflow y variar fuentes con Tailwind.

Pasos a seguir en resolved.html

  1. 1. Abrir `resolved.html` y trabajar sobre ese archivo.
  2. 2. Mantener el JavaScript de submit funcionando.
  3. 3. Hacer que el logo sea el primer hijo visual del `form`.
  4. 4. Centrar la tarjeta principal y darle rounded, sombra y fondo.
  5. 5. Alinear `label + input` en cada fila usando grid o flex.
  6. 6. Agregar placeholders claros y mantener `required`.
  7. 7. Estilizar el boton con color, hover y transicion suave.
  8. 8. Comprobar que exista `#login-message` y se muestre al enviar.

Paleta sugerida

Usa `bg-brand-soft`, `text-brand-dark`, `bg-brand-accent` y `hover:bg-brand-accentDark` para mantener una identidad visual consistente con Tailwind.

Criterios que validan los tests

  • Debe existir un fondo con color o gradiente definido.
  • Se esperan varias clases `rounded-*` en elementos visibles.
  • El `form` debe usar `id="login-form"`.
  • El logo debe quedar primero dentro del formulario.
  • Los campos deben incluir `Usuario`, `Contrasena`, placeholder y `required`.
  • Los labels y campos deben quedar alineados con clases responsivas.
  • El boton debe tener `transition` y hover con escala o elevacion.
  • Se esperan distintas fuentes o tamanos como `font-sans`, `font-serif`, `font-mono`, `text-4xl`.
  • Debe evitarse overflow horizontal normal.
  • Debe existir `id="login-message"` con mensaje correcto al enviar.
  • No se permite agregar `style.css`, `styles.css` ni otro stylesheet externo.

Regla importante

Si agregas un archivo aparte como `style.css`, `styles.css` o un `` distinto de Tailwind, los tests deben fallar con este mensaje:

NO USAR OTRO STYLE.CSS

Como hacer pasar los tests

1. Ejecuta `npm run test:jest -- tests/jest/junior1.test.js` para revisar la estructura esperada.

2. Ejecuta `npx playwright test tests/playwright/junior1.spec.js` para validar render, submit y responsividad.

3. Si una prueba falla, compara `resolved.html` con estos criterios y corrige clases, ids o textos.
