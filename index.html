YurtzCamp

Aprender programacion en equipo

YurtzCamp es una pagina de practica pensada para ensenar, aprender y compartir lo que fuimos descubriendo sobre Git, GitHub y desarrollo web en tiempo real.

Actividad 1

Trabajo en PP

Repaso de Git, Live Share, errores de terminal, ramas, origenes y despliegue. Esta actividad resume la primera practica compartida.

Bitacora

Aprendimos que Git no es solo para subir archivos, sino para ordenar el trabajo, colaborar mejor y recuperar el control del proyecto. Entendimos para que sirven git add, git commit, git push, git pull y git remote -v, y tambien por que --force puede ser peligroso.

Tambien aparecio el error 403, que nos ayudo a ver que Live Share comparte edicion y terminal, pero no permisos ni credenciales de GitHub. Ademas practicamos estructura de carpetas, servidor local, Emmet y trabajo en equipo con cambios en vivo.

Entre pruebas, risas y errores fuimos descubriendo como funciona el flujo real de trabajo. Vimos cambios en vivo, entendimos mejor la estructura de carpetas del proyecto y reforzamos el cuidado al ejecutar comandos desde la ruta correcta.

Satisfactory Bar 10/10

Actividad 2

Javascript en entornos Basicos

Vamos a usar un clon sin origen del repositorio AlhuayOscar/RestaurantMenuExample dentro de una carpeta aparte llamada EjemploRestaurante para estudiar como el JavaScript modifica el DOM, que etiquetas toca, como influye sobre el CSS y que comportamiento activa en la interfaz.

Bitacora

Esta actividad queda orientada a analizar el clon EjemploRestaurante para entender que hace el JavaScript, como toca el DOM y que clases o estilos impacta dentro del proyecto.

La idea es revisar eventos, selectores, cambios de clases, sliders, menus, efectos visuales y cualquier relacion directa entre JavaScript, HTML y CSS.

Guia del proyecto

Que queremos lograr

Roadmap activo

YurtzCamp quiere ser una base de estudio para formar a un desarrollador full stack capaz de disenar, construir, desplegar y mantener productos reales. La meta no es aprender tecnologia aislada, sino entender flujo de trabajo, arquitectura, colaboracion, UX, cloud, bases de datos y despliegue continuo.

Que hacemos

  • Practicar HTML, CSS, JavaScript y TypeScript.
  • Aprender Git, GitHub, terminal y trabajo colaborativo.
  • Registrar sesiones de pair programming y errores reales.
  • Convertir cada practica en una bitacora util.

Stack objetivo

  • MERN con React, Node.js, Express y MongoDB.
  • Next.js para apps full stack y despliegue en Vercel.
  • Prisma, PostgreSQL y Firebase segun el proyecto.
  • Redux Toolkit, Tailwind y Material UI para escalar.

Roadmap tecnico

  • Frontend: React, Next.js, CSS Modules, Framer Motion.
  • Backend: APIs, JWT, nodemailer, Stripe y PayPal.
  • Datos: Prisma, Sequelize, MongoDB, SQL y DynamoDB.
  • Cloud: Docker, AWS S3, Lambda y Google Console.

Vision

  • Ensenar a programar con ejemplos y contexto real.
  • Pasar de practica guiada a proyectos completos.
  • Desarrollar criterio tecnico y autonomia.
  • Construir una base fuerte para full stack moderno.
